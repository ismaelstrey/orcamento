## Objetivo

Explique o resultado esperado de forma direta.

## Escopo

- o que deve ser feito;
- o que deve ser atualizado;
- o que deve ser validado.

## Fora do escopo

- o que não deve ser alterado;
- o que não deve ser decidido nesta tarefa.

## Restrições

- usar TypeScript;
- não usar `any`;
- respeitar arquitetura em camadas;
- seguir convenções de nomes em camelCase;
- usar hooks para acesso à API no frontend;
- comentar apenas o necessário e em pt-BR.

## Saída esperada

- arquivos alterados;
- resumo das decisões;
- riscos remanescentes;
- testes ou validações feitas;
- pendências abertas.

## Critérios de aceite

- comportamento principal coberto;
- padrões do projeto respeitados;
- documentação atualizada quando necessário;
- sem divergência com ADRs vigentes.

# Documentacao

Este diretório concentra a documentação de planejamento do projeto.

## Objetivo

Criar uma base documental forte o suficiente para:

- orientar implementação humana;
- orientar implementação assistida por IA;
- reduzir retrabalho;
- manter consistência de produto, arquitetura e qualidade;
- servir como referência viva durante a evolução do SaaS.

## Convenções

- Todos os documentos estão em Markdown.
- O idioma principal é pt-BR.
- O planejamento prioriza clareza para uso por pessoas e agentes.
- Decisões críticas devem ser registradas em `docs/planning/adr/`.
- O código futuro deve seguir o que for aprovado nesta documentação.

## Áreas

- `planning/`: visão consolidada do produto, arquitetura, entrega, qualidade e roadmap.
- `planning/adr/`: registro de decisões arquiteturais e estratégicas.

## Regra operacional

Nenhum módulo relevante deve ser implementado sem que exista, no mínimo:

- objetivo claro;
- escopo definido;
- entidades envolvidas;
- fluxos principais;
- critérios de aceite;
- riscos conhecidos.

# ADR

## Objetivo

Registrar decisões arquiteturais e estratégicas que não devem ficar implícitas em conversas soltas.

## Quando criar um ADR

Criar um ADR quando uma decisão:

- altera a arquitetura;
- muda a estratégia de produto;
- impacta vários módulos;
- pode gerar divergência entre agentes;
- envolve trade-off importante.

## Estrutura recomendada

- contexto;
- decisão;
- consequências;
- alternativas consideradas;
- status.

## Primeiras decisões já consolidadas

- documentação antes de implementação;
- MVP sem dependência de IA;
- arquitetura monorepo com organização em camadas;
- núcleo tenant-aware desde o início.

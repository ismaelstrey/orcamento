# Delivery Workflow

## Objetivo

Definir como o projeto deve ser executado do planejamento até a entrega, com forte apoio de IA e preservação de qualidade.

## Modelo de trabalho

O projeto deve seguir um fluxo em camadas:

1. documento;
2. decisão;
3. contrato;
4. implementação;
5. validação;
6. atualização documental.

## Regra operacional

Nenhuma entrega deve começar direto no código quando houver lacuna em:

- escopo;
- regra de negócio;
- entidades;
- contrato;
- critérios de aceite.

## Macroetapas

### Etapa 1 - Fundacao

- consolidar PRD;
- aprovar MVP;
- definir arquitetura;
- registrar ADRs centrais;
- estruturar repositório.

### Etapa 2 - Core SaaS

- autenticação;
- tenant e usuários;
- clientes;
- produtos;
- categorias;
- orçamento;
- PDF;
- link público;
- histórico e versões.

### Etapa 3 - Fortalecimento

- dashboard útil;
- regras de compatibilidade;
- exportação/importação robusta;
- observabilidade;
- testes principais.

### Etapa 4 - Inteligencia

- IA para input e drafts;
- recomendações;
- comparação inicial;
- alternativas.

### Etapa 5 - Mercado e automacao

- preços;
- histórico;
- monitoramento;
- alertas;
- integrações externas;
- billing.

## Fatia ideal de execução

Cada fatia de trabalho deve:

- resolver um problema observável;
- atravessar as camadas necessárias;
- ser pequena o suficiente para revisão clara;
- terminar com documentação atualizada.

## Checkpoint obrigatorio antes de implementar

- objetivo definido;
- arquivos/documentos de referência identificados;
- dependências mapeadas;
- critérios de aceite escritos;
- risco principal conhecido.

## Checkpoint obrigatorio antes de encerrar

- comportamento validado;
- documentação atualizada;
- limitações registradas;
- próximos passos evidentes;
- impactos técnicos conhecidos.

## Politica para tarefas de IA/agentes

- tarefas pequenas e focadas;
- uma responsabilidade central por tarefa;
- saída verificável;
- revisão posterior em tarefas sensíveis;
- uso de contexto documental explícito.

## Fluxo recomendado por issue futura

1. abrir objetivo;
2. vincular documentos;
3. registrar aceite;
4. delegar implementação;
5. validar;
6. revisar;
7. atualizar documentação;
8. fechar.

## Definition of Done operacional

Uma entrega só deve ser considerada pronta quando:

- atende ao escopo prometido;
- segue padrões definidos;
- não viola decisões arquiteturais;
- possui validação mínima proporcional ao risco;
- deixa claro o que ficou pendente.
